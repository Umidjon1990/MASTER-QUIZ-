9) Qarzdorlarni aniqlash (logika)

Qarzdor = due_date o‘tgan + status != SUBMITTED

10) Replit AI uchun PROMPT PAKET (copy-paste)

Quyidagilarni Replit AI’ga ketma-ket bering. “Vibe coding” uchun juda qulay:

Prompt 1 — Project scaffold

Build a full-stack web app called “Task Tracker for Teachers”. Use Next.js (App Router) + TypeScript + Tailwind + shadcn/ui. Use Supabase for auth and Postgres database. Create pages: Dashboard, Groups, Group Details (Students, Lessons, Assignments, Tracker), Telegram Settings. Implement responsive UI.

Prompt 2 — Database schema

Create a Supabase Postgres schema with tables: groups, students, lessons, assignments, lesson_assignments, submissions, group_telegram, notifications_log. Add relations and indexes. Generate TypeScript types for queries.

Prompt 3 — Group creation wizard

Implement a “Create Group” wizard: group name, level, start date, schedule type (weekly days or every-other-day), number of lessons. On submit, auto-generate lessons with dates. Save to database. Show group in “My Groups”.

Prompt 4 — Assignments + linking to lessons

In group details, implement “Assignments” CRUD (title, description, due rule). Implement linking assignments to lessons and auto creating lesson_assignments with due_date.

Prompt 5 — Tracker grid

Build a Tracker page showing a grid: rows = students, columns = lesson_assignments. Each cell shows status + score. Clicking opens a modal to set: status (SUBMITTED/MISSING/REWORK/PENDING), score, feedback. Persist to submissions table. Optimize performance.

Prompt 6 — Debtors view

Implement a “Debtors” panel that filters submissions where due_date passed and status != SUBMITTED. Group by student and show missing tasks count. Add button “Send to Telegram”.

Prompt 7 — Telegram integration

Add Telegram Settings page: input bot token and chat_id per group. Add “Send test message”. Implement server action / API route to sendMessage using Telegram Bot API. Log sent messages to notifications_log.

Prompt 8 — Automation (optional)

Add a cron-like scheduled job (daily at 21:00) to send debtors list to each group Telegram automatically. Use a server-side scheduler compatible with deployment (Supabase edge functions / Vercel cron / Replit cron). Ensure it doesn’t spam.

Prompt 9 — Security and roles

Ensure teacher can only access their own groups/students/submissions. Add row-level security (RLS) policies in Supabase. Validate all inputs.
